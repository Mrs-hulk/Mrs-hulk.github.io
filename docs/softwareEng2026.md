# ソフトウェア工学で学んだこと
## ソフトウェアの定義

以下の3つから定義される．

1. 実行されることによって必要な特性，機能，性能を提供する命令語

2. プログラムが適切に情報を扱うことを可能とするデータ構造

3. プログラムの操作や使用法を記述した情報

## ソフトウェア工学の必要性

ハードウェアと異なり劣化はしないが，時代や技術により悪化はするため常にニーズを満たすように適応する必要がある．また，社会に欠かせないものの一部であるため，維持・開発するための人材育成も欠かせない．

## ソフトウェアライフサイクル

ソフトウェアの誕生から廃止までのことをソフトウェアライフサイクルと言う．ソフトウェア開発は， **計画**， **要件定義**， **設計**， **制作**， **テストとデバッグ**の流れで行われる．

## ソフトウェア分析

ソフトウェアを評価するために，物量や実行性能，開発工数，品質に関する分析を行う．評価には品質・費用・納期のどれを優先するか決める必要がある．

## Work Breakdown Structure

集団でプロジェクト目標をん達成するために，実行する作業などを階層的に要素分解して作成する．WBS作成後，すべての作業はWBSを参考に実行すればよい．

## コーディング

企業等で使用するコードは書くよりも読まれることの方が多い．それによりコーディングにルールを要する．PythonのPEP8などがある．
>GoogleにはGoogleのコーディング規約があり，読み手の思考コストを削減し，レビューや保守を効率化している．

コーディング規則を守っているか確認する **Linter**というものがある．
PEP8に基づくコードチェックツールはfleak8である．

## Github

リモートリポジトリのホスティングサービスの1つである．githubフローというものがあり，基本的にはそれに従い開発を行う．

### Codespaces

ブラウザ上でVScode環境を構築できる．ここでコマンドなどを使用してブランチを生み出したり，捜査することが出来る．

### CI/CD

**Continuous Integration(CI)**
>自動テストとビルドを手危機的に実行し，バグの早期発見と修正を可能にする．CIの自動化により開発のスムーズな進行を促進する．

**Continuous Delivery(CD)**
>コード変更をテスト環境や本番環境に自動的にデプロイするプロセスで，手動操作の必要性を排除する．CDにより，ユーザーフィードバッグを迅速に反映可能にする．

### Github Pages

Github上にあるリモートリポジトリの内容をWebページとして公開できるサービスのことである．Markdownファイルを作成するだけでJekyllが自動的にHTMLへ変換し，ホームページを公開できる．ホームページの公開はCI/CDにより自動化される．
